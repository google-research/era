# ERA: An AI System to Help Scientists Write Expert-Level Empirical Software

[![arXiv](https://img.shields.io/badge/arXiv-2509.06503-b31b1b.svg)](https://arxiv.org/abs/2509.06503)
[![Project Page](https://img.shields.io/badge/Project%20Page-google--research.github.io%2Fera-blue)](https://google-research.github.io/era/)

This repository contains code associated with the paper:

> **[An AI system to help scientists write expert-level empirical software](https://arxiv.org/abs/2509.06503)**
> Aygun, et al., 2025

## Overview

ERA (Empirical Research Assistant) is an AI system that helps scientists write
high-quality empirical software. It combines a large language model with a
tree-search algorithm — Flat UCB Tree Search (FUTS) — to iteratively generate,
execute, and score candidate programs, converging on expert-level solutions.

## Getting Started

### Prerequisites

- Python 3.10+
- A Gemini API key (set as `GOOGLE_API_KEY` environment variable)
- Required packages: `pandas`, `numpy`, `scikit-learn`, `google-generativeai`

```bash
pip install pandas numpy scikit-learn google-generativeai
```

### Running the Example

The `implementation/` directory contains a self-contained end-to-end example
(`playground_s3e1.py`) that applies ERA to a Kaggle-style regression task
(California Housing, Playground Series S3E1):

```bash
cd implementation/
export GOOGLE_API_KEY="your-api-key"
python playground_s3e1.py
```

Or open `experiment_pipeline.ipynb` in Jupyter to run a full experiment
interactively.

## Implementation: Flat UCB Tree Search (FUTS)

The `implementation/` directory contains a reference implementation of the Tree
Search algorithm described in Algorithm 1 of the paper.

### Example Task Notebooks

The `implementation/notebooks/` directory contains example task notebooks from
our evaluation benchmark, on two scientific tasks.
See [`implementation/notebooks/README.md`](implementation/notebooks/README.md)
for details.
